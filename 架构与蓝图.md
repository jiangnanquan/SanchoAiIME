# SanchoAiIME 架构与实现蓝图

## 1. 产品定位

**SanchoAiIME 是 AI 原生个人输入系统。**

它不是传统大词库输入法，也不是万象/Rime 的外挂增强器。Rime 只承担极薄的输入事件、候选 UI 和上屏职责；Sancho 负责个人记忆、本地小模型预测、云端大模型分析、快速字典、AI 命令和环境 profile 管理。

核心原则：

* **打字主链路绝不依赖网络**：本地小模型和基础词典必须足够完成日常输入。
* **大模型只做分析和蒸馏**：DeepSeek V4 Flash 等大模型不能进入按键实时链路。
* **用户喜好显式可控**：`/Users/jnq/Library/Rime/custom_phrase.txt` 是一等快速字典入口。
* **不污染本机环境**：AI 工具配置、skills、MCP、provider 和环境变量由 Sancho profile 托管。
* **词库是风格资产**：导入其他输入法词库后，不只是加词，还要提取领域、语气、短语长度和混写习惯。
* **大模型是维护工**：复杂归纳、计算、迁移、清理和蒸馏任务可以启动大模型完成，但必须异步、可审计、可回滚。
* **发布合规先行**：所有依赖、模型、词库、skills、图标、字体、配置模板和外部工具都必须有清晰许可证边界。

## 2. AI 分层

### Level 0: 输入主链路

职责：保证跟手、稳定、可离线。

```text
按键事件 -> 极简拼音/候选 -> 本地候选排序 -> 上屏
```

要求：

* 禁止同步网络请求。
* 禁止同步数据库写入。
* 候选 UI 优先显示基础候选，本地预测结果异步补充。

### Level 1: 本地小模型

默认基线：`Qwen3.5-0.8B`。

职责：

* 当前上下文候选重排。
* 下一词/短语预测。
* 轻量命令意图识别。
* 本地风格补全。

运行方式：

* 常驻本地服务，例如 MLX、llama.cpp 或 OpenAI-compatible local server。
* 输入为最近上屏文本、当前编码、候选列表、应用上下文。
* 输出必须是结构化候选，不允许自由聊天式输出。

### Level 2: 云端教师模型

默认目标：`DeepSeek V4 Flash API`。

职责：

* 分析历史输入，提取个人术语、项目名、人名、技术词。
* 给导入词库打领域和风格标签。
* 做词库瘦身，避免个人词库无限膨胀。
* 生成本地小模型可消费的规则、prompt、adapter 数据或候选权重。

约束：

* 只在用户授权的离线批处理里运行。
* 默认脱敏或按域过滤后上传。
* 产物必须回写到本地结构化文件或 DuckDB。

当前实现入口为 `packages/cloud-teacher` / `sancho-cloud-teacher`。DeepSeek
V4 Flash 凭据只能来自 `DEEPSEEK_API_KEY` 或 macOS Keychain service
`SanchoAiIME DeepSeek API Key`，CLI 只输出凭据是否可用及来源，不输出 Key
值。真实 API 请求需要显式 `--allow-network`。

### Level 3: 大模型维护工

Sancho 可以在复杂维护任务上启动大模型，形成可审计的后台 job。

适合任务：

```text
lexicon_summarize      从历史输入和导入词库归纳领域、风格和高价值短语
lexicon_merge          合并同码词、重复词、冲突权重和多来源词库
profile_migrate        迁移 AI CLI profile、MCP、skills 和 provider 配置
rule_synthesis         从用户习惯生成本地候选重排规则
distill_dataset        生成 Qwen3.5-0.8B 的本地训练/评测样本
health_audit           检查词库膨胀、危险命令、无效短码和低价值候选
explain_stats          把 DuckDB 统计转成用户可读的维护建议
```

job 结构：

```json
{
  "id": "job_20260427_lexicon_merge",
  "kind": "lexicon_merge",
  "model": "deepseek-v4-flash",
  "scope": ["rime_custom_phrase", "imported_sogou"],
  "privacy_mode": "redacted",
  "budget_cents": 10,
  "status": "pending_review"
}
```

执行要求：

* 不阻塞输入主链路。
* 用户可查看 prompt、输入范围、输出 diff 和费用预估。
* 默认只生成建议，用户确认后才写入快速字典、profile 或词库。
* 对所有写操作生成可回滚快照。
* 云端任务不得上传 secret、API Key、未授权剪贴板内容或敏感项目路径。

## 3. 快速字典与用户喜好

### 3.1 Rime 兼容底座

Sancho 必须支持并尊重：

```text
/Users/jnq/Library/Rime/custom_phrase.txt
```

这个文件是用户喜好的快速字典，格式保持 Rime 原生 tab 分隔：

```text
文本或短语	编码	权重
DuckDB	du	50
Playwright	pl	50
```

Sancho 不应该粗暴覆写整个文件。自动生成内容只能写入 marker 包裹的托管区：

```text
# >>> SanchoAiIME managed: quick-dictionary
SanchoExo Codex DeepSeek	cds	99
DeepSeek V4 Flash 分析	dsf	99
Qwen 本地预测	qwp	99
# <<< SanchoAiIME managed: quick-dictionary
```

用户手写区域永远保留。

### 3.2 Action Registry

`custom_phrase.txt` 只适合承载可见短语。真正的动作、命令、环境切换、skills 调用必须进入 Sancho action registry。

推荐结构：

```json
{
  "id": "sanchoexo.codex.deepseek",
  "code": "cds",
  "label": "SanchoExo + Codex + DeepSeek",
  "kind": "profile_switch",
  "profile": "sanchoexo-codex-deepseek",
  "insert_preview": "SanchoExo Codex DeepSeek",
  "risk": "normal"
}
```

动作类型：

```text
insert_text      插入普通文本
copy_text        复制到剪贴板
run_command      启动命令，但必须经用户授权
profile_switch   切换 AI 工具 profile
open_url         打开本地或 Web 控制台
skill_invoke     调用 Sancho/Codex/Claude skill
```

### 3.3 UI 引导

Dashboard 的快速字典页面必须让用户清楚知道每个短码会做什么。

页面分区：

* **短语**：文本、编码、权重、预览。
* **AI 命令**：命令名称、适用工具、是否需要确认。
* **Profile Switch**：provider、模型、base URL、MCP、skills、环境变量。
* **导入词库**：来源、去重策略、领域标签、写入目标。

安全提示：

* `run_command` 默认需要确认。
* API Key 等 secret 存 Keychain 或系统安全存储，不写进 `custom_phrase.txt`。
* Sancho 默认不修改 `.zshrc`、`.bashrc`、全局 `.env`。

## 4. Profile Switch

Sancho 需要实现类似 cc-switch 的能力，但不要把输入法变成配置编辑器。

cc-switch 的启发点：

* 多 AI CLI 工具统一管理。
* provider 一键切换。
* MCP 与 skills 集中配置。
* profile 可视化编辑。

Sancho 的落地方式：

```text
输入短码 -> 候选栏或 Dashboard 选择 -> Sancho wrapper 启动目标工具
```

示例 profile：

```json
{
  "id": "sanchoexo-codex-deepseek",
  "label": "SanchoExo / Codex / DeepSeek",
  "cwd": "/Users/jnq/Dev/Private/SanchoExo",
  "command": "codex",
  "args": [],
  "env": {
    "OPENAI_BASE_URL": "https://api.deepseek.com",
    "OPENAI_MODEL": "deepseek-v4-flash"
  },
  "skills": ["code-review", "web-research"],
  "mcp": ["github", "browser"]
}
```

环境变量注入策略：

* 默认只对 Sancho 启动的子进程生效。
* 用户明确选择“写入工具配置”时，才原子更新对应 CLI 的配置文件。
* 永不把 secret 写入 Rime 词库。

## 5. 外部词库导入

Sancho 必须支持流行输入软件的用户词库导入，作为冷启动和风格学习材料。

优先级：

* Rime: `dict.yaml`, `custom_phrase.txt`
* 搜狗输入法: `scel`, txt 导出
* QQ 拼音: 用户词库导出格式
* 百度输入法: 用户词库导出格式
* macOS/微软拼音: 可导出的用户词表或纯文本
* 通用 CSV/TSV

统一内部结构：

```json
{
  "surface": "DuckDB",
  "reading": "duck db",
  "weight": 90,
  "source": "rime",
  "domain": "data",
  "style_tags": ["tech", "english-mixed"],
  "last_used_at": null
}
```

导入后处理：

* 去重与权重合并。
* 中文、英文、代码符号分流。
* DeepSeek V4 Flash 离线打标签。
* 高频短语写入快速字典候选。
* 长尾词保留在 Sancho memory，不全部塞回 Rime。

## 6. 发布与许可证策略

SanchoAiIME 是要发布的软件，许可证策略必须作为架构约束，而不是发布前补丁。

### 6.1 项目许可证

建议自有代码使用 **Apache-2.0**：

* 允许商业和开源分发。
* 带专利授权条款。
* 与 Qwen3.5、Gemma 4 等 Apache-2.0 模型生态兼容。

如果未来决定闭源部分模块，必须把开源核心、闭源 Dashboard、模型权重和外部 GPL 工具拆成独立分发边界。

### 6.2 第三方组件边界

| 组件 | 许可证/约束 | 发布策略 |
| --- | --- | --- |
| `librime` | BSD-3-Clause | 可依赖，保留版权和许可证 |
| Squirrel/鼠须管 | GPL-3.0 | 默认不捆绑，只写用户配置；若捆绑或改造需完整 GPL 合规 |
| `imewlconverter` 深蓝词库转换 | GPL-3.0 | 不嵌入非 GPL 主程序；可做外部 CLI 适配或重写解析器 |
| Qwen3.5-0.8B | Apache-2.0 | 可选下载或分发，附 LICENSE/NOTICE |
| Gemma 4 E2B/E4B | Apache-2.0 | 可选下载或分发，附 LICENSE/NOTICE |
| DeepSeek V4 Flash API | 服务条款/API 计费 | 用户自带 API Key；不分发模型；遵守平台条款 |
| 用户导入词库 | 用户数据 | 不上传、不再分发；只在本地转换和分析 |

### 6.3 禁止事项

* 不把 GPL 代码复制进 Apache-2.0 主程序。
* 不把 API Key 写入 `custom_phrase.txt`、Rime YAML、日志或前端 bundle。
* 不随软件打包搜狗、QQ、百度等第三方商业词库。
* 不默认上传完整输入历史；云端分析必须有显式授权和脱敏选项。
* 不发布来源不明的图标、字体、提示词库、skills 或模型量化包。

### 6.4 Release Gate

每次 release 前必须生成并检查：

```text
LICENSE
NOTICE
THIRD_PARTY_NOTICES.md
MODEL_LICENSES.md
SBOM
dependency license scan
model artifact manifest
GPL isolation checklist
privacy/data-flow checklist
```

如果某个依赖许可证不清晰，默认不进入发布包。

## 8. 实施路径

### Phase 1: 快速字典最小闭环

* 读取 `/Users/jnq/Library/Rime/custom_phrase.txt`。
* 识别用户手写区和 Sancho 托管区。
* 生成/更新托管区，但不破坏用户原有条目。
* 建立 `sancho-actions.json` 的最小 action registry。

### Phase 2: Profile Switch

* 定义 profile schema。
* 实现 `sancho exec --profile <id>` wrapper。
* UI 支持创建 Codex/Claude/OpenCode/Gemini profile。
* 环境变量只注入子进程。

### Phase 3: 本地模型预测

* 接入 `Qwen3.5-0.8B`。
* 建立候选重排 benchmark。
* 确定输入法可接受延迟阈值。

### Phase 4: DeepSeek 教师分析

* 接入 DeepSeek V4 Flash API。
* 分析历史输入和导入词库。
* 输出词库标签、候选权重、风格 profile。

### Phase 5: 词库导入

* 先支持 Rime 与通用 TSV。
* 再接入搜狗、QQ 拼音、百度、macOS/微软拼音。
* UI 提供预览、筛选、冲突处理和回滚。

### Phase 6: 大模型维护工

* 建立 job queue 和审计日志。
* 实现词库合并、profile 迁移、健康检查三个首批任务。
* 支持查看大模型输出 diff，确认后写入。

### Phase 7: 发布合规

* 确定项目 LICENSE。
* 生成第三方 NOTICE 与模型许可证清单。
* 建立 GPL 隔离规则和 release gate。
* 为用户数据、云端分析和 API Key 存储写隐私说明。
