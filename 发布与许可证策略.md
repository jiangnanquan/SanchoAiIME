# SanchoAiIME 发布与许可证策略

本文是 SanchoAiIME 的发布合规基线。实现功能时如果和本文冲突，以更保守的合规策略为准。

本文不是法律意见。正式商业发布前，应由项目所有者或法律顾问确认最终许可证、商标、模型权重分发和第三方服务条款。

## 推荐许可证

项目自有代码建议采用 **Apache License 2.0**。

理由：

* 对商业发布友好。
* 明确包含专利授权。
* 和 Qwen3.5、Gemma 4 等 Apache-2.0 模型生态兼容。

## 组件分发边界

| 类型 | 例子 | 策略 |
| --- | --- | --- |
| 自有代码 | Sancho backend, dashboard, CLI, Lua probe | Apache-2.0 |
| Rime 引擎 | `librime` | BSD-3-Clause，可依赖并保留 NOTICE |
| Rime macOS 宿主 | Squirrel/鼠须管 | GPL-3.0，默认不捆绑，只生成配置 |
| 词库转换器 | `imewlconverter` | GPL-3.0，不嵌入主程序，可外部 CLI 适配 |
| 本地模型 | Qwen3.5, Gemma 4 | 按模型许可证分发，附 LICENSE/NOTICE |
| 云端模型 | DeepSeek V4 Flash API | 用户自带 API Key，遵守服务条款 |
| 用户词库 | 搜狗/QQ/百度/Rime 导入文件 | 用户本地数据，不随软件再分发 |

## GPL 隔离规则

Sancho 主程序不复制、链接或内嵌 GPL-3.0 代码，除非整个相关分发单元按 GPL-3.0 合规发布。

允许：

* 提示用户安装鼠须管。
* 生成 Rime 配置文件和 Lua 脚本。
* 调用用户系统里已安装的外部 CLI。
* 为 GPL 工具提供“路径配置”和“命令行适配器”。

不允许：

* 把 Squirrel.app 打进 Sancho 安装包后忽略 GPL 源码义务。
* 复制 `imewlconverter` 的 GPL 解析代码进 Apache-2.0 主程序。
* 使用 GPL 词库/方案作为默认内置资源而不满足对应许可证。

## 模型许可证规则

发布包可以提供模型下载器，而不是默认捆绑全部模型权重。

如果捆绑模型权重，必须：

* 记录模型名称、版本、来源 URL、hash、许可证。
* 随包提供模型 LICENSE/NOTICE。
* 标注是否允许商用、微调、再分发、量化再分发。
* 区分官方权重和第三方量化权重。

默认模型：

```text
实时预测基线：Qwen3.5-0.8B
本地备选：Gemma 4 E2B/E4B
云端教师：DeepSeek V4 Flash API
```

## API 与隐私

DeepSeek 等云端 API 只能用于用户授权的后台任务。

要求：

* API Key 存 Keychain 或系统安全存储。
* 不写入 Rime 词库、日志、前端 bundle、截图或崩溃报告。
* 上传前提供范围选择和脱敏。
* 记录每个任务的模型、输入范围、费用估算和输出 diff。
* 默认不上传完整输入历史。

## 词库导入规则

Sancho 只处理用户提供的词库文件，不随软件分发第三方商业输入法词库。

导入后产生的数据属于用户本地数据：

* 可以写入 DuckDB。
* 可以生成用户自己的 `custom_phrase.txt` 托管区。
* 不能作为 Sancho 默认公开词库再发布。

## Release Gate

每次 release 前必须检查：

```text
LICENSE 是否存在
NOTICE 是否存在
THIRD_PARTY_NOTICES.md 是否更新
MODEL_LICENSES.md 是否更新
SBOM 是否生成
GPL 组件是否被错误嵌入
API Key 是否可能泄露
测试数据是否包含真实用户输入
导入词库样例是否可再分发
```

任何一项不确定，默认移出发布包。
